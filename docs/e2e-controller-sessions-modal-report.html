3. Networking: Make Frontend and Backend Communicate Reliably

Must Be Correct For Production
Problem Why It Happens Production Fix
Mixed content (HTTPS frontend calling HTTP backend) Vercel serves HTTPS. Browsers block insecure requests/WS from secure origins. Put TLS on backend (Nginx/Caddy) and use https://api.yourdomain.com in rewrites / CONTROLLER_BASE_URL.
CORS failures Backend must explicitly allow the Vercel origin(s). Set ALLOW_ORIGINS and optionally ALLOW_ORIGIN_REGEX for preview deployments.
WebSocket upgrade failures via reverse proxy Proxy config missing Upgrade/Connection headers. Enable WS proxying in Nginx/Caddy for /controller/proxy/sessions/*/ws/*.
Modal sandbox cannot call controller heartbeat CONTROLLER_BASE_URL must be reachable from Modal (public network), not localhost. Set CONTROLLER_BASE_URL=https://api.yourdomain.com.
Controller cannot reach Modal tunnel Sandbox process crashes before listening (seen in E2E due missing pgvector). Fix Modal image dependencies and sandbox env propagation, then re-test /controller/proxy/.../healthz.

Firewall / Ports (Backend Server)

# Example with ufw
sudo ufw allow 22/tcp
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# If temporarily exposing uvicorn directly (not recommended long term):
sudo ufw allow 8000/tcp

Nginx WebSocket Example (controller proxy)

location / {
    proxy_pass http://127.0.0.1:8000;
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}

location /controller/proxy/sessions/ {
    proxy_pass http://127.0.0.1:8000;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
    proxy_set_header X-Forwarded-Proto $scheme;
}
